## From `code-analyze-2026-04-16.md`

### ¬ß6.1 ‚Äî `query` vs `scan` boundary not enforced by types

~~The `query.*` namespace documents O(1)/O(log n) operations and `scan.*` documents O(n), but nothing prevents a caller from using `scan.*` inside a hot rendering loop. A lint rule or `@complexity` JSDoc tag on the `scan` namespace would help.~~

**Partial fix applied (2026-05-29):** The `scan.*` module-level comment now carries a prominent `@remarks` warning ("Do not call `scan.*` inside a hot rendering loop or on every keystroke") with a `@see query` reference and explicit guidance on appropriate vs. inappropriate use. Full enforcement via a lint rule remains a future work item.

| ¬ß6.4 ‚Äî Chunk streaming API is imperative and easy to sequence incorrectly | Added `StreamingDocumentLoader` (`src/store/features/streaming-loader.ts`) exported as `createStreamingDocumentLoader` from `reed`.
